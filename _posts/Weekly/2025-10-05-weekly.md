## Codex v.s. Claude Code

Codex 在解決單點問題上似乎比 Claude 給力了？模型比較聰明的關係？今天讓 Claude（Opus） 處理一個 Jinja2 模板的問題，浪費了一堆 token 仍沒搞定，Codex 一個對話就修好了。也許之後要嘗試協調不同設備上的 coding agent 合作辦事。不過 Codex 有時還是有點蠢笨 - 我把我的配色方案（含一段 CSS 設定）給 Codex 叫它據此改造我的 Flask app，結果它寫了個函數去 parse 那個文件裡面的 CSS snippet！到底是在幹啥，徒增複雜度。

小筆記：要在遠端機上啟用 codex 需要把 1455 端口映射到本地，`ssh -L 1455:127.0.0.1:1455 user@server`。
